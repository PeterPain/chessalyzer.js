Class: Chessalyzer

Chessalyzer

Main class for batch processing and generating heat maps

new Chessalyzer()

Members

dataStore :Array.<Object>

Contains the tracked data of the processed PGN files. Has two different banks for heat map comparison. Each object contains the following keys:
  • data: {cntMoves, cntGames}. Information about the count of processed moves and games
  • tiles: 8x8 array of ChessTiles.
Type:
  • Array.<Object>

Methods

generateComparisonHeatmap(square, fun [, bank1] [, bank2])

Generates a comparison heatmap out of the tracked data. There needs to data in both banks you pass as bank1 and bank2 params. The heatmap for both banks are calculated and then the relative differences between both banks are calculated. For example, if the heatmap value for "a1" of bank1 is 10 and the value of bank2 is 5, the returned value for "a1" would be 100% ([[10/5] -1] *100).
Parameters:
Name Type Argument Default Description
square String The square the data shall be generated for.
fun function The evaluation function that generates the heatmap out of the saved data. See Chessalyzer#generateHeatmap for a more detailed description.
bank1 Number <optional>
 0 Bank number of dataset 1
bank2 Number <optional>
 1 Bank number of dataset 2
Returns:
Array with 3 entries:
  1. 8x8 Array containing the heat map values for each tile
  2. The minimum value in the heatmap.
  3. The maximum value in the heatmap.

generateHeatmap(bank, square, fun)

Generates a heatmap out of the tracked data.
Parameters:
Name Type Description
bank Number The data bank the data shall be taken from
square String The square the data shall be generated for. For example, if you wanted to know how often a specific piece was on a specific tile, you would pass the identifier of the tile to the function, e.g. "a2"
fun function The evaluation function that generates the heatmap out of the saved data. This function gets passed the following arguments:
  1. The complete data stored in the chosen bank. See the member description of the dataStore member to see which data is available.
  2. The coords of the tile passed as the square argument.
  3. The current coordinates of the tile the data should be generated for. The function must return a Number with the heat map value for the square passed as the third argument.
See ./src/exampleHeatmapConfig for examples of such a function.
Returns:
Array with 3 entries:
  1. 8x8 Array containing the heat map values for each tile
  2. The minimum value in the heatmap.
  3. The maximum value in the heatmap.
Type
Array
